기본 정보

기본 정보 작성 기준 Table, Category, Entity는 필수입니다.
새 파일 저장 시 카테고리-엔티티-테이블.md 형식의 파일명으로 사용됩니다.
Category
파일 구조에서 가장 먼저 묶을 큰 분류입니다. 예: member, order
Entity
Category 안에서 테이블을 묶는 업무 대상입니다. 예: User, Payment
Table
실제 DB 테이블명을 적습니다. 예: users, order_items
Description
테이블의 목적, 주요 사용 흐름, 운영 메모를 한곳에 적습니다.

컬럼 정의

컬럼 정의 작성 기준
한글명
업무에서 부르는 컬럼 이름입니다.
화면, 기획서, 엑셀 명칭을 맞출 때 씁니다.
Column
실제 DB 컬럼명입니다.
인덱스, 제약조건, SQL 생성에서 이 값을 기준으로 참조합니다.
Type
DB 컬럼 타입입니다. 예: varchar, bigint, timestamp
Length
문자열 길이나 숫자 자릿수처럼 타입에 필요한 크기입니다. 예: 255
PK / FK
기본키와 외래키 여부입니다.
복합키나 FK 상세 관계는 제약조건에 적습니다.
Nullable
NULL 허용 여부입니다.
체크하면 NULL을 허용하는 컬럼으로 기록됩니다.
Unique
단일 컬럼 중복 불가 여부입니다.
여러 컬럼 조합의 Unique는 제약조건에 적습니다.
Default
DB 기본값입니다. 예: now(), 0, false
Example
문서에서 참고할 값 예시입니다. 예: user@example.com
설명
컬럼의 의미와 사용 목적을 적습니다.
규칙
입력 형식, 허용 범위, 코드 규칙을 적습니다.
메모
마이그레이션, 운영 주의사항, 예외 케이스를 적습니다.

인덱스 정의

인덱스 정의 작성 기준 인덱스는 자주 조회하거나 정렬하는 컬럼의 검색 속도를 높이기 위해 작성합니다.
작성 시점
WHERE, JOIN, ORDER BY에 자주 쓰이는 컬럼이 있을 때 작성합니다.
Index Name
DB에 생성할 인덱스 이름입니다. 예: idx_users_email
Columns
컬럼 정의의 Column 값을 선택합니다.
입력하면 자동완성으로 후보가 나오고, 여러 컬럼은 칩으로 묶입니다.
Unique
중복을 막는 인덱스인지 표시합니다.
단순 중복 방지인지 조회 성능 목적 인덱스인지 구분할 때 씁니다.
Sort
정렬 방향이 SQL에 필요할 때 적습니다. 예: created_at DESC
설명
어떤 조회, 화면, 배치 작업을 위해 필요한지 적습니다.
메모
특수 인덱스, 운영 주의사항, 성능 메모를 적습니다.

제약조건

제약조건 작성 기준 컬럼 하나로 표현하기 어려운 관계, 복합키, 복합 유니크, 조건 규칙을 기록합니다.
요약 행은 종류 / 제약명 / 설명만 보이고, 화살표를 누르면 상세 입력이 열립니다. 항목 설명
종류
FK, Composite PK, Unique, Check 중 하나를 선택합니다.
선택하면 필요한 상세 입력 항목이 아래에 나타납니다.
제약명
DB에 등록할 제약조건 이름입니다. 예: fk_orders_user_id
설명
요약 행에 함께 보이는 짧은 설명입니다.
왜 필요한 제약인지 적습니다.
대상 컬럼
현재 테이블에서 제약이 걸리는 Column입니다.
컬럼 정의에 있는 값만 선택할 수 있고, 여러 개면 칩으로 묶어서 관리합니다.
참조 테이블
FK일 때 참조하는 대상 테이블입니다. 예: users
참조 컬럼
FK일 때 참조하는 대상 컬럼입니다.
쉼표 앞뒤 공백은 정리되어 id,user_id와 id, user_id 모두 id, user_id로 저장됩니다.
On Update
FK 참조값이 바뀔 때의 정책입니다.
화살표로 기본 옵션을 고르고, Custom은 같은 필드 안에서 직접 입력합니다.
On Delete
FK 참조 데이터가 삭제될 때의 정책입니다.
화살표로 기본 옵션을 고르고, Custom은 같은 필드 안에서 직접 입력합니다.
조건
Check일 때 값의 허용 범위나 조건식을 적습니다. 예: amount > 0
메모
마이그레이션, 운영 주의사항, DB별 차이처럼 설명과 분리해 둘 내용을 적습니다.
종류별 사용법
FK
현재 테이블 컬럼이 다른 테이블 컬럼을 참조할 때 씁니다.
복합 FK는 대상 컬럼과 참조 컬럼을 같은 순서로 적고, 서로 다른 참조 관계가 여러 개면 제약조건을 여러 개 만듭니다.
Composite PK
두 개 이상의 컬럼을 묶어 하나의 기본키로 사용할 때 씁니다.
Unique
여러 컬럼 조합의 중복을 막을 때 씁니다.
단일 컬럼 중복 금지는 컬럼 정의의 Unique로도 충분합니다.
Check
값의 범위나 형식 같은 조건을 DB 레벨에서 제한할 때 씁니다.
FK 액션 옵션
NO ACTION
DB 기본 검사 시점에 따라 참조 무결성을 확인합니다.
RESTRICT
참조 중인 데이터의 변경이나 삭제를 즉시 막습니다.
CASCADE
부모 데이터 변경이나 삭제를 자식 데이터에도 전파합니다.
SET NULL
부모 데이터가 바뀌거나 삭제되면 자식 컬럼을 NULL로 바꿉니다.
SET DEFAULT
부모 데이터가 바뀌거나 삭제되면 자식 컬럼을 기본값으로 바꿉니다.
Custom
DB나 프로젝트에서 쓰는 별도 정책명을 직접 입력합니다.
