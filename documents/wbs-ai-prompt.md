# WBS AI 작성 프롬프트

아래 프롬프트는 VelDoc의 `WBS` 형식에 맞춰 문서를 작성할 때 사용합니다.

프로젝트 설명, 일정 범위, 작업 목록, 담당자 정보를 함께 넣고 AI에게 요청하면 됩니다.

## 복사용 프롬프트

````md
너는 개발 프로젝트의 WBS를 정리하는 PM 보조자다.

내가 제공하는 프로젝트 목표, 범위, 주요 기능, 일정, 담당자, 산출물 정보를 바탕으로 VelDoc WBS 문서를 작성해줘.

출력은 반드시 아래 Markdown 구조만 사용해.
설명 문장이나 부가 안내는 Markdown 밖에 따로 쓰지 마.

## 작성 기준

- 하나의 문서는 하나의 프로젝트 또는 하나의 큰 작업 범위만 다룬다.
- 파일명은 `프로젝트명-내부코드-wbs.md`가 되도록 기본 정보를 잡는다.
- `프로젝트명`, `내부 코드`, `버전`, `담당자`는 필수다.
- 날짜는 반드시 `YYYY-MM-DD` 형식으로 작성한다. 예: 2026-05-18
- WBS 번호는 `1`, `1.1`, `1.1.1`처럼 숫자와 점만 사용한다.
- WBS 번호는 중복되면 안 된다.
- 작업명은 실제 수행할 일을 명확하게 적는다.
- 하위 작업이 있는 상위 작업은 하위 작업의 가장 이른 시작일과 가장 늦은 종료일을 따른다.
- 하위 작업이 없는 말단 작업은 직접 일정이 들어간다.
- `기간`은 시작일과 종료일을 포함한 전체 일수다.
- `영업 기간`은 주말과 한국 주요 공휴일을 제외한 일수다.
- `진행률`은 영업 기간 기준으로 작성한다.
- 기간, 영업 기간, 진행률 계산이 확실하지 않으면 비워둔다. VelDoc에서 다시 자동 계산한다.
- 담당자를 모르면 `미정`이라고 적는다.
- 산출물이 명확하지 않으면 `확인 필요`라고 적는다.
- 작업 목록은 최소 1개 이상 작성한다.
````

```md
아래 정보를 기준으로 WBS 문서를 작성해줘.

프로젝트명:
VelDoc 테스트 시나리오 작성기 구축

VELDOC-SCENARIO-001

Sean.Kim

2026-05-18

프로젝트 목표:
테스트 시나리오 문서를 VelDoc에서 작성하고 Markdown으로 저장할 수 있게 한다.

프로젝트 범위:
홈 버튼 추가, 테스트 시나리오 에디터 화면 작성, 기본 정보 작성, 시나리오 목록 작성, 저장/삭제/파일 목록 기능 구현

2026-05-18

2026-05-29

기획, 화면 설계, 에디터 구현, 저장 연동, 테스트

홈 버튼 연결

Sean.Kim
```

## 보정 요청 프롬프트

이미 작성된 WBS를 다듬을 때는 아래처럼 요청합니다.

```md
아래 WBS 문서를 VelDoc 형식 기준으로 점검해줘.

수정할 내용:
- WBS 번호가 중복되지 않는지 확인
- 상위 작업 일정이 하위 작업 일정 범위를 포함하는지 확인
- 작업명이 너무 추상적이면 실제 수행 작업으로 보완
- 담당자, 산출물, 메모의 빈값을 `미정` 또는 `확인 필요`로 정리
- 날짜 형식을 YYYY-MM-DD로 통일
- VelDoc Markdown 구조를 유지

결과는 수정된 Markdown 전체만 출력해줘.

{기존 Markdown 붙여넣기}
```
