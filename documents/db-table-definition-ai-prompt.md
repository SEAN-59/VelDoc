# DB 테이블 정의서 AI 작성 프롬프트

아래 프롬프트는 VelDoc의 `DB 테이블 정의서` 형식에 맞춰 문서를 작성할 때 사용합니다.

필요한 테이블 정보나 업무 설명을 함께 넣고 AI에게 요청하면 됩니다.

## 복사용 프롬프트

````md
너는 개발 문서를 정리하는 DB 설계 보조자다.

내가 제공하는 업무 설명, 화면 흐름, API 정보, 기존 컬럼 정보, ERD 메모를 바탕으로 VelDoc DB 테이블 정의서를 작성해줘.

출력은 반드시 아래 Markdown 구조만 사용해.
설명 문장이나 부가 안내는 Markdown 밖에 따로 쓰지 마.

## 작성 기준

- 하나의 문서는 하나의 테이블만 다룬다.
- 파일명은 `카테고리-엔티티-테이블.md`가 되도록 기본 정보를 잡는다.
- `Category`는 가장 큰 업무 분류다. 예: member, order, payment
- `Entity`는 Category 안에서 테이블을 묶는 업무 대상이다. 예: User, Order, Payment
- `Table`은 실제 DB 테이블명이다. snake_case를 우선 사용한다.
- 컬럼의 `Column` 값은 실제 DB 컬럼명이다.
- `PK`, `FK`, `Nullable`, `Unique`는 반드시 `Y`, `N` 중 하나를 고른다.
- `Nullable`의 `Y`는 NULL 허용, `N`은 NOT NULL을 뜻한다.
- FK 상세 관계는 컬럼 정의의 FK만으로 끝내지 말고 `제약조건`에도 작성한다.
- 복합 PK, 복합 Unique, 복합 FK는 `제약조건`에 작성한다.
- 인덱스와 제약조건의 대상 컬럼은 `컬럼 정의`의 Column 값을 정확히 사용한다.
- 여러 컬럼은 쉼표와 공백으로 구분한다. 예: user_id, role_id
- FK의 `On Update`, `On Delete`는 `NO ACTION`, `RESTRICT`, `CASCADE`, `SET NULL`, `SET DEFAULT` 중 하나를 우선 고른다.
- 판단이 어려운 항목은 비워두지 말고 `확인 필요`라고 적는다.
- SQL은 위에 작성한 기본 정보, 컬럼, 인덱스, 제약조건을 기준으로 생성한다.
````

## 요청 예시

```md
아래 정보를 기준으로 DB 테이블 정의서를 작성해줘.

서비스 / 업무:
회원 관리

테이블 목적:
사용자 계정 정보를 저장한다.

카테고리:
member

엔티티:
User

예상 테이블명:
users

주요 기능:
회원 가입, 로그인, 사용자 조회, 계정 비활성화

저장해야 하는 데이터:
이메일, 비밀번호 해시, 이름, 휴대폰 번호, 상태, 마지막 로그인 일시

필수 컬럼:
id, email, password_hash, name, status, created_at, updated_at

관계가 있는 테이블:
현재 없음

조회 조건 / 정렬 조건:
email로 단건 조회한다.
status별 목록 조회가 있다.
created_at 기준 최신순 정렬이 있다.

삭제 정책:
논리 삭제

개인정보 / 민감정보 여부:
이메일, 이름, 휴대폰 번호는 개인정보다.
비밀번호 해시는 민감한 보안 데이터다.

추가 메모:
email은 중복되면 안 된다.
```

이미 작성된 정의서를 다듬을 때는 아래처럼 요청합니다.

```md
아래 DB 테이블 정의서를 VelDoc 형식 기준으로 점검해줘.

수정할 내용:
- 누락된 컬럼 설명 보완
- 인덱스와 제약조건이 컬럼 정의와 맞는지 확인
- SQL이 문서 내용과 일치하는지 확인
- 불필요하거나 과한 항목은 간단하게 정리

결과는 수정된 Markdown 전체만 출력해줘.

{기존 Markdown 붙여넣기}
```
